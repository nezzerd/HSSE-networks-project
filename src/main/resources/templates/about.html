О проекте

Search Engine — учебный мини-поисковик, написанный на Spring Boot. Он самостоятельно обходит сайты, складывает страницы в базу, строит по ним полнотекстовый указатель и позволяет искать по нему через браузер или API.

Как это работает

Система состоит из трёх частей, работающих внутри одного приложения:

Краулер
Робот обходит сайты в ширину (BFS), скачивает HTML-страницы, извлекает текст и ссылки. Соблюдает robots.txt, делает паузы между запросами к одному домену и не выходит за заданные лимиты.
Индексатор
Берёт скачанные страницы и строит по ним инвертированный указатель на Apache Lucene: для каждого слова — список страниц, где оно встречается. Поддерживает русский и английский языки.
Поиск
По запросу пользователя обращается к указателю, ранжирует результаты алгоритмом BM25 и показывает их с подсвеченными фрагментами текста.

Как пользоваться

Введите запрос на главной странице — результаты появятся на отдельной странице со сниппетами и постраничной навигацией.

Также доступен REST API: запрос вида /api/search?q=ваш+запрос вернёт результаты в формате JSON. Полное описание API — на странице документации.

Безопасность и устойчивость

Сервис спроектирован так, чтобы выдерживать некорректные и злонамеренные обращения:

Защита от SSRF
Краулер не ходит на внутренние и приватные адреса; каждый переход по редиректу проверяется отдельно.
Защита от XSS
Текст с чужих страниц экранируется при выводе — встроенный в результаты код не может выполниться в браузере пользователя.
Ограничение частоты запросов
Поиск и управление краулером ограничены по числу запросов с одного адреса, что защищает от перегрузки.
Лимиты обхода
Ограничены размер страницы, глубина, общее число страниц и размер очереди — робот не может работать бесконечно или переполнить хранилище.

Технологии

Java 21, Spring Boot 3, Thymeleaf, PostgreSQL (H2 для разработки), Apache Lucene, jsoup. Приложение упаковано в Docker и работает за HTTPS-прокси Caddy. Серверный рендеринг страниц делает сайт доступным для индексации поисковыми системами.
